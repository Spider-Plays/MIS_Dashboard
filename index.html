📐 Excel Formula Reference — Recruiter Performance Overview click to collapse
📋 Positions Assigned
=SUMPRODUCT(
  ISNUMBER(MATCH(MRF[Hire Category],
    {"Lateral","Consultant","Fresher","Contract"},0))*
  ISNUMBER(MATCH(MRF[Quarter],activeQuarters,0))*
  (MRF[Req Type]="Child Req")*
  (MRF[Recruiter]="RecruiterName")
)
Uses: Valid Hire Cats · Child Req Type · Active Quarters
🔄 Active Positions
=SUMPRODUCT(
  ISNUMBER(MATCH(MRF[Hire Category],
    {"Lateral","Consultant","Fresher","Contract"},0))*
  (MRF[Req Type]="Child Req")*
  (MRF[Requisition Status-CR]="Sourcing")*
  (MRF[Approval Status]="Approved")*
  (MRF[Recruiter]="RecruiterName")
)
Uses: Valid Hire Cats · Child Req · Status-CR=Sourcing · Approval=Approved
👥 Profiles Sourced
=COUNTIFS(
  CandSheet[Hire Category], {"Lateral",...},
  CandSheet[Recruiter], "RecruiterName"
)
-- Dashboard uses: all valid-hire-cat candidates
   for this recruiter in active slicer context
From Candidate_Sheet · filtered by current slicers
📤 Offers & ✅ Joiners
Offers = COUNTIFS where
  Sub Status ∈ MIS_OFFER_STATUSES

Joiners = COUNTIFS where
  Sub Status = MIS_JOINED_STATUS

Both filtered by recruiter + active slicers
Uses: MIS Offer Statuses · MIS Joined Status below
⏱ Avg Aging of Requisitions (days)
=IFERROR(AVERAGE(FILTER(
  MRF[Req Aging (in Days)],
  (MRF[Req Type]="Child Req")*
  (MRF[Quarter]<>"")*
  (MRF[Requisition Status-CR]="Joined")*
  (MRF[Req-Month]<>"")*
  (MRF[Recruiter]="RecruiterName")*
  ISNUMBER(MATCH(MRF[Hire Category],{"Lateral","Consultant","Fresher","Contract"},0))*
  ISNUMBER(MATCH(MRF[Quarter],activeQuarters,0))
)),"—")
Column needed: Req Aging (in Days) in MRF sheet · Status-CR must be "Joined" · Quarter & Req-Month must be non-blank
MIS Analytics uses separate formula constants (MIS_INTERVIEW_STATUSES, MIS_OFFER_STATUSES etc.) that exactly mirror your Excel MIS formulas — they may differ from the main status groups above. MIS also uses Candidate Sub Status field (configurable in Column Mapping) and filters by Valid Hire Categories and Child Req type.
🧮 MIS Interview Statuses
Exact statuses from your Excel formula that count as "interviewed" in MIS recruiter scorecards. One per line, exact match.
🧮 MIS Offer Statuses
Exact statuses from your Excel formula that count as "offered" in MIS. Note: Excel typically includes 'joined' here since you can't join without an offer.
🧮 MIS Joined Status
Single status value meaning "joined" for MIS (usually just "joined").
🧮 MIS Screen Reject Status
Single status value for screen reject in MIS (usually "screen reject").
🏷 Valid Hire Categories
MIS only counts rows whose Hire Category matches one of these values. One per line, exact match.
🏷 Child Req Type Value
MIS filters MRF rows where "Req Type" column equals this value (usually "Child Req").
Open Position: Status-CR Value
MRF Status-CR value meaning "actively being sourced" (usually "Sourcing").
Open Position: Approval Value
Approval Status value that must also match for Open Positions count (usually "Approved").
👤 Recruiter Table — Visible Names
Enter recruiter names to show in the Recruiter Performance Overview table — one per line. Leave blank to show all. Names are matched case-insensitively (partial match).
How it works:
• Blank = all recruiters shown
• Partial match: "Pravin" matches "Pravin Christo"
• Names matched case-insensitively
• Order of rows = order in table
• Use Auto-detect to load all recruiter names found in your current data
📈 Experience Buckets
Format: Label | min | max — one per line. Last bucket use Infinity as max. Min is inclusive, max is exclusive.
Example:
0–2 yrs | 0 | 3
3–5 yrs | 3 | 6
6–8 yrs | 6 | 9
9–12 yrs | 9 | 13
13+ yrs | 13 | Infinity
